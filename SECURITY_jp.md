# OpenRAGセキュリティポリシーと責任ある情報開示

## セキュリティポリシー

このセキュリティポリシーは、GitHub上のlangflow-ai組織のすべての公開プロジェクトに適用されます。私たちはセキュリティを最優先に考え、システムの保護に継続的に取り組んでいます。ただし、脆弱性が存在する可能性はあります。セキュリティ上の問題を発見した場合は、迅速に対処できるようご報告ください。

### セキュリティおよびバグ修正バージョン

- 修正は次のマイナーバージョン（例：1.3.0 → 1.4.0）の一部として、またはオンデマンドパッチバージョン（例：1.3.0 → 1.3.1）としてリリースされます
- セキュリティ修正は優先的に対応され、新しいバージョンのリリースが必要になる場合があります

## 脆弱性の報告

セキュリティ脆弱性の責任ある情報開示を推奨しています。セキュリティ上の問題を発見または疑われた場合は、迅速に対処できるよう、慎重にご報告ください：

### レポートの提出

[OpenRAG セキュリティページ](https://github.com/langflow-ai/openrag/security)にアクセスし、**Report a vulnerability（脆弱性を報告）**をクリックして、あなたとリポジトリのメンテナー間のプライベートな会話を開始してください。

問題を迅速に再現・修正するために、以下を含む具体的な詳細をできる限り提供してください：

- 問題を再現する手順
- 潜在的な影響または懸念事項
- 推奨される修正方法

あなたのレポートは機密として扱われ、あなたの同意なしにこれらの詳細は共有されません。

### 対応タイムライン

報告を受け取ってから5営業日以内に確認のご連絡をいたします。

解決の見込み時期をお知らせします。

進捗状況についてご報告いたします。

### 情報開示のガイドライン

- 脆弱性を評価・解決し、影響を受けるユーザーへの通知が完了するまで、公開での情報開示はご遠慮ください。
- 研究成果を発表する予定がある場合（例：カンファレンスやブログ）、レビューのために少なくとも30日前にドラフトをご共有ください。
- 情報開示には以下の内容を含めないでください：
  - OpenRAGのカスタマープロジェクトのデータ
  - OpenRAGのユーザー/カスタマー情報
  - OpenRAGの従業員、請負業者、またはパートナーに関する詳細

セキュアなプラットフォームの維持にご協力いただいていることに感謝申し上げます。問題を責任ある形で解決するために、皆様と協力できることを楽しみにしています。

## セキュリティ設定ガイドライン

### 認証を有効にしてLangflowサーバーを起動する

Langflowのパスワード（`LANGFLOW_SUPERUSER_PASSWORD`）を設定することで、Langflowサーバーが認証を有効にした状態で起動し、`langflow superuser` コマンドが無効化されることが推奨されます。

このパスワードはOpenRAGのインストール時に設定するか、[OpenRAGの `.env` ファイルを編集してOpenRAGコンテナを再デプロイ](https://docs.openr.ag/reference/configuration#set-environment-variables)することで設定できます。

詳細については、[OpenRAGのLangflow設定リファレンス](https://docs.openr.ag/reference/configuration#langflow-settings)をご覧ください。
