# ウェブサイト

このウェブサイトは、モダンな静的サイトジェネレーターである[Docusaurus](https://docusaurus.io/)を使用して構築されています。

## インストール

```bash
npm install
```

## ローカル開発

```bash
npm start
```

このコマンドはローカル開発サーバーを起動し、ブラウザウィンドウを開きます。ほとんどの変更はサーバーを再起動しなくてもリアルタイムで反映されます。

## ビルド

```bash
npm run build
```

このコマンドは静的コンテンツを `build` ディレクトリに生成し、任意の静的コンテンツホスティングサービスで提供できます。

## デプロイ

SSHを使用する場合：

```bash
USE_SSH=true npm run deploy
```

SSHを使用しない場合：

```bash
GIT_USER=<GitHubユーザー名> npm run deploy
```

GitHubページでホスティングしている場合、このコマンドはウェブサイトをビルドして `gh-pages` ブランチへプッシュする便利な方法です。

## OpenRAGドキュメントPDFの更新

`openrag/openrag-documents/openrag-documentation.pdf` にあるドキュメントPDFはOpenRAGアプリケーションで使用されるため、常に最新の状態に保ってください。

PDFを更新するには、以下の手順を実行します：

1. `docs/*.mdx` ファイルから要素を削除します。
タブ、details、およびsummary要素内のコンテンツはPDFビルドでは非表示になるため、含める必要があります。
これらの要素を削除するには、IDEに以下のプロンプトまたは類似のものを与えてください。

   ```
   PDFのためのドキュメントのフラット化：タブとdetails要素を削除
   docs/docs/ 内のすべてのMDXファイルで、インタラクティブ要素をフラット化する：
   すべての <Tabs> と <TabItem> コンポーネントを削除：
   各タブのコンテンツを適切な見出しを持つ通常のセクションに変換する（サブセクションは###、メインセクションは##）
   すべてのタブコンテンツを順番に表示
   不要になったTabsとTabItemのインポート文を削除
   すべての <details> と <summary> 要素を削除：
   detailsコンテンツを適切な見出しを持つ通常のテキストに変換する（サブセクションは###）
   すべてのコンテンツを直接表示（折りたたみセクションなし）
   すべてのコンテンツを表示状態に保つ — 何も非表示や折りたたまれた状態にしない
   適切なフォーマットと構造を維持する
   PDF生成のためにコンテンツが完全にフラットで見える状態になるよう、タブまたはdetails要素を含むすべてのドキュメントファイルに適用する。
   ```

2. `.mdx` ファイルを確認して、これらの要素が削除されていることを確認します。
変更をコミットしないでください。

3. `openrag/docs` から次のコマンドを実行して、変更を含むサイトをビルドし、`openrag/openrag-documents` にPDFを作成します。

   ```
   npm run build:pdf
   ```

4. PDFの内容を確認してから、コミットしてプルリクエストを作成します。
